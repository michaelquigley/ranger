Decisions worth knowing:

- **Drop-indicator snapping is semantic, not visual.** A drop in the unranked tail *serializes* as end-of-ranked-list per the work order, and a dashed rule marks each lane's ranked/unranked boundary — but dnd-kit's live preview still shows the card at the pointer position during a tail-region hover rather than snapping the indicator to the boundary. Faithful to the write model, one visual notch short of the work-order sentence; flagging for soak review rather than building custom collision detection speculatively.
- **No live cross-lane re-parenting during drag** (no `onDragOver` container moves): the drop resolves the gesture and the board repaints from the server's fresh board. Less preview fidelity, much less state to get wrong.
- **One outcome path in the client**: every mutation resolves to `ok | conflict | invalid | fault`. Board-level conflicts notice-and-reload; `slug_collision` and validation stay local to the capture modal / item panel, which keep their content on screen; faults show the server's message verbatim.
- **Within-lane reorder computes the resulting ranked prefix client-side** (`rankedAfterMove`) and PUTs only that — dragging an unranked card ranks it alone; a no-op move sends nothing.
- The vite dev proxy targets `127.0.0.1:4114` (vane's own default), so `npm run dev` + `vane serve` is the hot-reload loop.
- `ui/dist/` and `ui/node_modules/` are gitignored; `make build` builds the frontend before the Go binary, per the work order's committed-builds-not-required call.
