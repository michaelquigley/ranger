# 2026-07-22 — live board reload

The `live-board-reload` roadmap item (spike), continuing yesterday's single-agent session. The mechanism seam went to Michael with three candidates — poll, fsnotify+SSE push, focus-refresh-only — and he took the recommendation: **poll, 2 seconds**. The push path was rejected as against the grain (a watcher lifecycle and subscription state on a server that deliberately holds nothing, plus SSE sitting awkwardly beside ogen); polling is the system's own idiom — every request already re-reads the disk, so freshness just gains a clock. No server or contract changes at all; the feature is entirely client-side.

- `ui/src/live.ts`: `POLL_MS = 2000` and `freshest(prev, next)` — the identity-preserving application (JSON-equality; unchanged reads keep the previous object) so a quiet tick repaints nothing.
- `App.tsx`: a `tick` that runs `reload()` — which already carries the index via `refreshIndex`, so dirty markers and availability ride the same clock — gated on `document.visibilityState` and a new `dragActive` ref. The ref guards at *application* time too (both the board set and the error set), so a poll that raced a drag start is dropped, never stomping the optimistic board. `visibilitychange` listens with the same tick: returning to a hidden tab is an immediate refresh — exactly the moment manual refresh used to serve. Poll failure applies the body-region error truthfully (same tier as a manual refresh finding it) and heals on the next tick. `refreshIndex` also goes identity-preserving.
- Tests: `live.test.ts` pins the freshest contract (identity kept, replacement on change, first-read adoption). 36 UI tests; full Go suite + vet + `make build` green.
- Docs: ui-and-serve.md's freshness contract rewritten (the manual-refresh sentence retired), README's board paragraph likewise; CHANGELOG FEATURE under Unreleased.

## npm audit: the js-yaml override

Michael flagged 2 high-severity findings in the ui dependencies. One root cause: `js-yaml` 4.2.0 (GHSA-52cp-r559-cp3m, quadratic CPU on YAML merge-key chains) reached through `@redocly/openapi-core` 1.34.17 under `openapi-typescript` — dev-tooling only (`gen:api`), nothing in the shipped bundle, so real exposure was nil (the only YAML it ever parses is ranger's own spec). `npm audit fix` was a no-op because redocly 1.34.17 pins js-yaml *exactly* at 4.2.0 and no fixed 1.x exists — the fix lives in redocly 2.x, which openapi-typescript 7.x (current latest) doesn't accept. Resolution: a scoped override in `ui/package.json` forcing js-yaml to 4.3.0 (the patched 4.x, same major as the pin). Audit now clean; `gen:api` regenerates `schema.d.ts` byte-identical; 36 UI tests + build green. Removal trigger captured as roadmap item `drop-the-js-yaml-override` (inbox): drop when openapi-typescript moves to redocly 2.x. No changelog entry (no behavior change) and no terminus pass (no product code touched — lockfile, package.json, and a roadmap item).
