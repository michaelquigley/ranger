# 2026-07-14 — stage 2 landed (implementation session, continued)

Stage 2 (document layer) built: `internal/document` with the two-pass parse, malformed-as-verdict item classification, order-document ops, guarded writes, and the exact-output fixture suite (49 tests). Deps added: `gopkg.in/yaml.v3` (this package only), `github.com/michaelquigley/df` (dd).

Decisions worth knowing:

- **Scalars decode as source text** in pass 2 (`nodeToPlain`), not through yaml.v3's type resolution — otherwise unquoted `created: 2026-07-13` arrives as `time.Time` and quoted arrives as `string`, and shape validation must judge the source text either way. dd still does per-field shape validation via `dd.Bind` on a one-field map, which keeps malformed-as-verdict (one broken sibling never erases another).
- **Line spans trim trailing blank/comment lines**, except block scalars keep comment-looking lines (indented `# ...` could be scalar content). Keep-chomping (`|+`) trailing blanks are accepted as out-of-posture.
- **Flow-style lanes** (`building: [a.md, b.md]`) materialize to block form when an op targets them — a hand can legally write flow, and the alternative was corrupting the file on a line-targeted edit. Same treatment for a lane whose whole value is an alias (`researching: *pool`).
- **Unknown-block prune guard is coarse**: one recognized alias into unknown territory keeps *every* unknown block that write. Unknown blocks may alias each other, so either all prune together (cross-references die together) or none do — a partial prune could dangle a transitive alias (terminus round 5 caught exactly that against the earlier per-block version).
- **Patches refuse on missing/duplicated fields** (`SetState` with no readable `state:` line) — repair for those goes through raw save, not a guessed insertion.
- `FinalizeLink` (link + remove src) is the no-clobber funnel; `CompareAndWrite` handles the absent sentinel with `O_CREATE|O_EXCL`.
- Synthesis in `docs/current/document.md`; CHANGELOG entry added.
